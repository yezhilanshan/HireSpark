### 技术栈

| 层级 | 技术 | 作用 |
|-----|------|------|
| **前端** | Next.js 15 + React 18 + TypeScript | 用户界面、实时交互 |
| **通信** | Socket.IO + WebSocket | 实时双向通信、自动重连 |
| **后端** | Flask + Flask-SocketIO | Web框架、事件处理 |
| **AI** | Qwen (DashScope SDK) | 问题生成、回答评估、简历解析 |
| **数据库** | SQLite3 | 面试记录、对话历史、简历存储 |
| **工具** | Pillow、JSON、Report生成 | 图片处理、数据处理、报告生成 |

### **2. LLM管理 (utils/llm_manager.py)**

**职责**: 与Qwen API集成，处理问题生成、回答评估、简历解析

**关键方法**:

```python
class LLMManager:
    
    def generate_round_question(self, round_type, position, difficulty):
        """生成初始问题
        参数:
          - round_type: 'technical'|'project'|'system_design'|'hr'
          - position: 'java_backend'|'frontend'|...
          - difficulty: 'easy'|'medium'|'hard'
        返回:
          - 初始问题字符串
        实现:
          1. 根据round_type选择对应的system_prompt
          2. 构造prompt调用Qwen
          3. 返回解析后的问题
        """
    
    def set_interview_round(self, round_type, resume_data=None):
        """设置面试轮次上下文
        用于后续问题和评估使用该轮次的prompt
        """
    
    def process_answer_with_round(self, user_answer, current_question, 
                                   position, round_type, chat_history=[]):
        """🌟 核心方法 - 处理回答并生成追问
        参数:
          - user_answer: 用户的回答
          - current_question: 当前问题
          - position: 职位
          - round_type: 轮次 (决定评估prompt)
          - chat_history: 之前的对话历史 (上下文)
        返回:
          - feedback: {
              "follow_up_question": "追问内容",
              "evaluation": "评价",
              "suggestions": ["建议1", "建议2"]
            }
        实现:
          1. 根据 round_type 选择对应的评估prompt
          2. 构造包含chat_history的完整prompt
          3. 调用 Qwen API
          4. 解析响应为结构化数据
          5. 返回追问和评价
        """
    
    def evaluate_answer(self, user_answer, question, position):
        """评估答案质量
        返回:
          - {
```

```
  │     │         ├─→ DashScope API 调用 (Qwen3-VL-Flash)
  │     │         │   └─→ 发送prompt + 图片
  │     │         │   └─→ 返回JSON {projects, experiences, skills, ...}
  │     │         └─→ 返回结构化数据
  │     │
  │     ├─→ DatabaseManager.update_resume_status(resume_id, 'parsed')
  │     │     └─→ UPDATE resumes SET status = 'parsed', parsed_data = {...}
  │     │
  │     └─→ HTTP Response 200
  │           └─→ { resume_id, data: parsed_result, success: true }
  │
  └─→ 前端显示解析结果
```

```
class SocketClient {
  static getInstance() {
    if (!this.instance) {
      this.instance = new SocketClient()

  }
  
  async connect() {
    // 连接到 ws://localhost:5000
    // 自动重连(5次): 失败 → 等待1秒 → 重试
    // 传输降级: WebSocket → HTTP Long-polling
  }
  
  on(eventName, callback) {
    // 监听服务器事件
  }
  
  emit(eventName, data) {
    // 发送数据到服务器
  }
}

// ========== 使用示例 ==========
const socket = SocketClient.getInstance()
await socket.connect()

// 发送事件
socket.emit('start_interview', {
  round: 'technical',
  position: 'java_backend',
  difficulty: 'medium'
})

// 监听事件
socket.on('interview_started', (data) => {

socket.on('llm_response', (data) => {
```

## 🎯 关键要点总结

### **1. 三个最核心的流程**

1. **面试流程** (`start_interview` → `llm_process_answer`(多轮) → `end_interview`)
2. **LLM处理** (`LLMManager.process_answer_with_round()` - 轮次化prompt管理)
3. **数据持久化** (内存 → 数据库 → 报告)

### **2. 关键的函数调用链**

```
前端 emit  
  ⬇️
后端 @socketio.on() 事件处理  
  ⬇️
业务逻辑 (DataManager, LLMManager)  
  ⬇️
数据库 (DatabaseManager)  
  ⬇️
响应 emit() 回前端  
```

### **3. 速率限制**

- **答案提交**: 5次/秒 (`answer_rate_limiter`)
- **面试操作**: 2次/10秒 (`interview_rate_limiter`)

### **4. 四轮面试的轮次区别**

| 轮次 | 考察重点 | Prompt特性 |
|------|--------|----------|
| `technical` | 基础知识、编码能力 | 偏理论、基础 |
| `project` | 项目经验、深度 | 基于简历、提升难度 |
| `system_design` | 架构、权衡 | 高阶思维 |
| `hr` | 软技能、规划 | 行为评估 |

### **5. 模块间的依赖关系**

```
app.py (主程序)
  ├─→ DataManager (内存数据)
  ├─→ LLMManager (AI处理) → DashScope API
  ├─→ DatabaseManager (数据库)
  │     └─→ SQLite
  ├─→ ReportGenerator (报告)
  ├─→ ResumeParser (简历) → DashScope API
  ├─→ RateLimiter (限流)
  └─→ Logger (日志)
```
