# 评价体系再升级实施记录

本文档用于记录本轮围绕《[评价体系再升级.md](d:/code/tianshuzhimian/职跃星辰/评价体系再升级.md)》已经完成的代码改动，方便后续回看、联调和继续迭代。

## 1. 本轮目标

本轮不是继续堆报告展示，而是把底层评分逻辑逐步推进到：

- 主判断
- 证据校正
- 稳定聚合

并且保持当前报告主结构兼容，不强行替换用户已经熟悉的顶部总分口径。

本轮实际落地范围包括：

- `Round Aggregation Layer` 第一阶段
- 内容轴从“规则混合”向“LLM 主锚点 + 证据校正”迁移
- `content / speech / video` 三轴 `confidence` 结构统一
- 将新的稳定聚合和可信度信息接入报告页

未在本轮实现的部分包括：

- 新建 `round_evaluations / interview_profiles` 表
- 用稳定分直接替换顶部总分
- 题型难度校准 `difficulty_index / calibrated_score`
- 完整的 `InterviewProfileBuilder`
- 把单题评语生成彻底后移到后端

## 2. 已完成改动总览

### 2.1 Round Aggregation Layer 第一阶段

已新增独立聚合模块，对单题评分结果做轮次级稳定化和轻量校准。

#### 已实现能力

- `RoundStabilizer`
  - 单题状态归一化
  - 基于 `confidence` 的基础权重
  - 基于中位数偏离的异常题降权
- `CrossTurnCalibrator v1`
  - 按 `round_type` 查询近期历史记录
  - 计算 `baseline_avg_score`
  - 输出 `relative_position / relative_band`
- 轮次聚合输出
  - `round_profiles`
  - `round_summary`
  - `interview_stability`

#### 状态映射规则

- `ok -> full_ok`
- `partial_ok` 且 `layer2.evaluation_mode == layer2_without_layer1_rubric -> partial_ok`
- `partial_ok` 且存在 fallback/error 回退痕迹 -> `fallback_only`
- `failed / skipped / review_required / veto -> excluded`

#### 权重规则

- `full_ok = 1.0`
- `partial_ok = 0.75`
- `fallback_only = 0.35`
- `excluded` 不进入主均值

#### 异常值抑制规则

- `deviation > 25 -> suppression_factor = 0.4`
- `deviation > 18 -> suppression_factor = 0.7`
- 否则 `suppression_factor = 1.0`

### 2.2 内容轴改造：LLM 主锚点 + 证据校正

原先内容轴更偏向：

- `LLM 分`
- `+ checklist/rubric 主评分参与`

本轮已改成：

- `LLM anchor score`
- `+ evidence correction`

也就是说，RAG/checklist 现在更接近“证据校正器”，而不是第二个近似平权的主评分器。

#### 新增结构

内容轴现在新增了 `evidence_correction`，包括：

- `llm_anchor_score`
- `positive_bonus`
- `missing_penalty`
- `risk_penalty`
- `net_correction`
- `corrected_score`
- `strategy = llm_anchor_plus_evidence_correction_v1`

### 2.3 三轴 confidence 统一

本轮已将：

- 内容轴
- 表达轴
- 镜头轴

统一到相同风格的 `confidence_breakdown` 结构。

#### 每一轴当前统一包含

- `data_confidence`
- `model_confidence`
- `rubric_confidence`
- `overall_confidence`

说明：

- `data_confidence`：样本是否足够、时长是否足够、有效信号是否充分
- `model_confidence`：模型推理链路、ASR/视觉检测稳定度
- `rubric_confidence`：规则/证据/命中点是否足够支撑当前判断
- `overall_confidence`：用于最终融合的总体可信度

#### 融合层改动

`fuse_layer_scores()` 现在会优先使用：

- `layer.confidence_breakdown.overall_confidence`

并且融合结果中已新增：

- `axis_confidence_breakdowns`

相关文件：

- [backend/utils/evaluation_service.py](d:/code/tianshuzhimian/职跃星辰/backend/utils/evaluation_service.py)

## 6. 当前仍未完成的部分

以下内容是文档中提到、但本轮还没有实现的重点：

### 6.1 RAG 进一步降权为“证据层”

虽然内容轴已经从“规则混合”向“主判断 + 证据校正”迈进了一步，但还没有完全完成：

- 报告层语义上还没有彻底去掉“checklist 像主评分来源”的感觉
- RAG 证据在报告和解释层仍可继续弱化为“支持证据”

### 6.2 题型/难度校准

当前只做了：

- 同轮次历史基线对比

还没有做：

- `question_type`
- `difficulty_index`
- `calibrated_score`
- `relative_position within question family`

### 6.3 Interview Profile Builder

当前报告仍以：

- 单题结果
- 轮次稳定化
- 多轴可信度

为主，还没有真正产出：

- `ability_profile`
- `stability_profile`
- `growth_profile`

### 6.4 单题解读后移到后端

目前单题“更像真实评语”的生成仍主要在前端拼装，未来更稳的方向是：

- 后端直接输出结构化摘要
- 前端只负责展示
