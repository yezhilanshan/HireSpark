**标题**：为什么需要 PanelMind？—— 四大求职痛点

**页面内容（四宫格布局）**：

## PPT 第 6 页：创新亮点一 —— 三层语音处理链路

**标题**：快与准兼得 —— 三层语音处理架构

**页面内容（三层架构图）**：

```
┌─────────────────────────────────────────────────────────┐
│  第一层：实时交互层                                      │
│  低延迟 ASR 字幕（延迟 < 500ms）                         │
│  自适应静音检测，动态调整触发阈值                         │
└─────────────────────────┬───────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  第二层：聚合展示层                                      │
│  题目级答案拼接与去重                                     │
│  实时构建用户回答的完整上下文                               │
└─────────────────────────┬───────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  第三层：终稿确认层                                      │
│  高精度 ASR 复写用于最终评估                               │
│  确保评分依据的准确性                                     │
└─────────────────────────────────────────────────────────┘
```

**技术参数**：
- 16kHz 采样率 | 回声消除 | 噪声抑制 | 自动增益控制
- 阿里云 DashScope Paraformer 实时语音识别
- 阿里云 CosyVoice 语音合成

**演讲备注（1分钟）**：
> "传统方案在实时字幕与最终评估之间难以兼顾：实时字幕要求低延迟，最终评估要求高准确率。我们创新性地设计了三层语音处理架构。第一层实时交互层，使用阿里云 DashScope Paraformer 实现低延迟 ASR 字幕，延迟控制在 500ms 以内。第二层聚合展示层，进行题目级答案拼接与去重，构建完整上下文。第三层终稿确认层，使用高精度 ASR 复写用于最终评估，确保评分依据准确。同时支持 16kHz 采样率、回声消除、噪声抑制和自动增益控制，适配多种面试环境。"

## PPT 第 7 页：创新亮点二 —— RAG 检索增强评分

**标题**：评分有据可依 —— RAG 检索增强与 LLM 协同

**页面内容（对比图）**：

```
传统方案                          PanelMind 方案
───────────                      ─────────────
纯 LLM 自由评分                   RAG + LLM 协同评分
    ↓                                 ↓
波动大、黑盒                      一致性高、可解释
不可追溯                         可引用知识库来源
```

**核心机制**：
1. 基于 **ChromaDB** 构建岗位知识库，支持语义 + 关键词混合检索
2. 评估时以检索结果作为"依据层"，提升评分一致性与可解释性
3. AI 问答助手同样基于 RAG，回答可引用知识库来源

**演讲备注（1分钟）**：
> "传统的大模型评分存在波动性和黑盒问题。PanelMind 创新性地引入 RAG 检索增强评分机制。我们基于 ChromaDB 构建岗位知识库，支持语义加关键词混合检索。评估时，系统以检索结果作为依据层，提升评分一致性与可解释性，避免纯大模型自由评分导致的波动性问题。AI 问答助手同样基于 RAG，回答可引用知识库来源，增强可信度。"

## PPT 第 8 页：创新亮点三 —— 三层评分与短木板调节

**标题**：诊断到点子上 —— 三层评分与短木板调节机制

**页面内容（评分维度表 + 短木板示意图）**：

| 维度 | 权重 | 评估指标 |
|------|------|---------|
| **内容层** | 70% | 技术正确性、知识深度、逻辑严谨性、岗位匹配度 |
| **表达层** | 20% | 语速、停顿、流畅度、清晰度 |
| **状态层** | 10% | 情绪稳定度、专注度、行为规范性 |

**短木板规则**：
> 关键维度低于阈值时，整体分数按比例下调，避免"偏科"被掩盖。

```
示例：
内容层得分：85（权重 70%）→ 59.5
表达层得分：60（权重 20%）→ 12.0  ⚠️ 低于阈值，触发短木板调节
状态层得分：80（权重 10%）→ 8.0

原始总分：79.5
调节后总分：72.3（表达层短板导致整体下调）
```

**演讲备注（1分钟）**：
> "PanelMind 采用三层评分体系。内容层占 70%，评估技术正确性、知识深度、逻辑严谨性和岗位匹配度。表达层占 20%，评估语速、停顿、流畅度和清晰度。状态层占 10%，评估情绪稳定度、专注度和行为规范性。特别值得一提的是短木板调节机制：当关键维度低于阈值时，整体分数会按比例下调，避免'偏科'被掩盖，确保用户全面发展。"

## PPT 第 12 页：技术栈与工程实践

**标题**：主流技术栈 —— 确保工程质量与可扩展性

**页面内容（三列布局）**：

**前端**
- Next.js 15 (App Router) + React 19 + TypeScript
- Tailwind CSS + CSS Variables（支持深色模式）
- Motion（交互动画）
- D3.js（知识图谱）+ Recharts（数据图表）
- 性能优化：代码分割、懒加载、Skeleton 加载态

**后端**
- Flask + Socket.IO（实时通信）
- 阿里云 DashScope（Qwen LLM + CosyVoice TTS + Paraformer ASR）
- RAG 检索增强（ChromaDB 向量数据库）
- SQLite（关系数据）
- 安全：Rate Limiting、输入校验、JWT 认证

**工程亮点**
- 模块化设计：服务层、工具层、数据层清晰分离
- 容错机制：各核心模块独立导入，单点故障不影响整体
- 配置驱动：YAML 配置文件管理，支持多环境部署
- 性能监控：内置性能监控与日志系统

**演讲备注（1分钟）**：

## 附录：答辩高频问题与推荐回答

### Q1：和普通聊天机器人有什么区别？
**A**：我们不是简单对话，而是 Agent 系统。不同的 Agent 负责不同职能（提问/追问/评估/报告），有上下文流转和状态管理，面试有节奏（开场→深入→收尾），评分有体系（多维度 + 可解释）。

### Q2：评分是不是主观？
**A**：系统采用"结构化指标 + 规则策略 + 模型解释"组合，不追求绝对客观，但追求可解释、可复核、可改进。RAG 检索增强让评分有知识库依据，避免纯模型自由评分的波动性。

### Q3：为什么必须做 RAG？
**A**：仅靠模型内置知识存在时效与幻觉问题。RAG 能让回答锚定在项目知识库并提供引用，可信度更高，评分一致性更好。

### Q4：项目最难点是什么？
**A**：不是页面开发，而是评估口径稳定性、多模态数据质量和三层语音处理链路的实时性与准确性平衡。

### Q5：这个项目有产品化空间吗？
**A**：有，场景可扩展到高校就业指导、职业培训、企业内训等多个方向，核心能力具备迁移价值。

## 附录：PPT 制作建议

### 视觉风格建议
- **主色调**：深色背景（#101217）+ 白色文字 + 青色（#06b6d4）强调色
- **字体**：中文使用 Noto Sans SC / Noto Serif SC，英文使用 Inter / Newsreader
- **图表**：使用 Recharts 风格的简洁图表，避免复杂装饰
- **图标**：统一使用 Lucide 图标体系

### 每页设计原则
- **一页一主题**：每页只讲一个核心观点
- **数据可视化**：能用图表不用表格，能用表格不用文字
- **留白充足**：避免信息过载，每页不超过 6 个要点
- **演讲备注**：每页底部预留演讲备注区，写清要讲的关键话术

### 动画建议
- 流程图使用逐步出现动画
- 数据卡片使用淡入动画
- 避免过度花哨的转场效果
