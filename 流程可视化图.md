## 2️⃣ LLM处理核心流程

```
用户回答 (user_answer)
    │
    ├─→ 输入验证 (validate_string)
    │     ✓ 长度检查 (1-5000)
    │     ✓ 特殊字符检查
    │
    ├─→ 选择 System Prompt (基于 round_type)
    │     ├─ round_type = 'technical'
    │     │  └─→ technical_system_prompt (基础知识导向)
    │     │
    │     ├─ round_type = 'project'
    │     │  └─→ project_system_prompt (项目经验导向)
    │     │
    │     ├─ round_type = 'system_design'
    │     │  └─→ system_design_prompt (架构设计导向)
    │     │
    │     └─ round_type = 'hr'
    │        └─→ hr_system_prompt (软技能导向)
    │
    ├─→ 构造完整Prompt
    │     ├─ System: [选中的prompt]
    │     ├─ History: [chat_history] (之前的对话)
    │     ├─ Current Question: [current_question]
    │     └─ User Answer: [user_answer]
    │
    ├─→ 调用 Qwen API (DashScope)
    │     ├─ Model: qwen-max 或 qwen3-vl-flash
    │     ├─ Temperature: 0.7 (创意与稳定的平衡)
    │     └─ Max tokens: 2000
    │
    ├─→ 解析 JSON 响应
    │     ├─ follow_up_question: "下一个问题"
    │     ├─ evaluation: "评价文本"
    │     └─ suggestions: ["建议1", "建议2"]
    │
    ├─→ 保存到内存 (DataManager)
    │     ├─ timeline.append({
    │     │    'type': 'interview_dialogue',
    │     │    'question': ...,
    │     │    'answer': ...,
    │     │    'feedback': ...,
    │     │    'timestamp': ...
    │     │  })
    │
    ├─→ 保存到数据库 (SQLite)
    │     ├─ INSERT INTO interview_dialogues (
    │     │    interview_id, round_type, question, answer, feedback
    │     │  )
    │
    └─→ 返回前端 (emit)
          ├─ llm_response 事件
          ├─ 包含追问 + 评价
          └─ 前端显示
```

## 3️⃣ 简历上传与解析流程

```
用户上传文件 (PDF/DOCX/PNG)
    │
    ├─→ 前端验证
    │     ├─ 文件类型检查
    │     ├─ 大小检查 (< 100MB)
    │     └─ 确认后上传
    │
    ├─→ HTTP POST /api/resume/upload
    │     ├─ FormData: {file, user_id}
    │     │
    │     └─→ 后端处理
    │           ├─ 验证文件
    │           ├─ 保存到 uploads/resumes/
    │           ├─ 计算 MD5 hash
    │           ├─ DB.save_resume() → status: 'parsing'
    │           │
    │           ├─→ ResumeParser.parse_file(file_path)
    │           │     ├─ 识别文件类型
    │           │     │
    │           │     ├─ 格式转换 (转为图片)
    │           │     │  ├─ PDF → pdf2image
    │           │     │  ├─ DOCX → python-docx + 转图
    │           │     │  └─ 图片 → 直接使用
    │           │     │
    │           │     ├─→ ResumeParser._parse_image(image_path)
    │           │     │     ├─ 读取图片
    │           │     │     ├─ Base64 编码
    │           │     │     │
    │           │     │     ├─→ DashScope API (Qwen3-VL-Flash)
    │           │     │     │     ├─ 模型: qwen3-vl-flash (视觉理解)
    │           │     │     │     ├─ 提示词: 
    │           │     │     │     │  "从简历中提取以下信息: ..."
    │           │     │     │     │
    │           │     │     │     └─ 返回 JSON 格式:
    │           │     │     │        {
    │           │     │     │          "projects": [
    │           │     │     │            {
    │           │     │     │              "name": "项目名",
    │           │     │     │              "technologies": ["技术"],
    │           │     │     │              "description": "描述"
    │           │     │     │            }
    │           │     │     │          ],
    │           │     │     │          "experiences": [
    │           │     │     │            {
    │           │     │     │              "company": "公司",
    │           │     │     │              "position": "职位",
    │           │     │     │              "duration": "时间",
    │           │     │     │              "description": "描述"
    │           │     │     │            }
    │           │     │     │          ],
    │           │     │     │          "skills": ["技能1", "技能2"]
    │           │     │     │        }
    │           │     │     │
    │           │     │     └─ 返回结构化数据
    │           │     │
    │           │     └─ 返回 parsed_data
    │           │
    │           ├─ DB.update_resume_status(resume_id, 'parsed')
    │           ├─ 将 parsed_data 保存到数据库
    │           │
    │           └─ HTTP Response 200
    │                 └─ {
    │                      "resume_id": 123,
    │                      "data": parsed_data,
    │                      "success": true
    │                    }
    │
    ├─→ 前端轮询检查解析状态
    │
    └─→ 解析完成
          ├─ 前端显示项目、经历、技能
          └─ 用户可编辑/删除
```

## 4️⃣ 报告生成流程

```
面试结束 (handle_end_interview)
    │
    ├─→ DataManager.export_for_report()
    │     ├─ 收集 timeline 数据
    │     ├─ 收集 events 数据
    │     └─ 返回完整的 interview_data
    │
    ├─→ ReportGenerator.generate_report(interview_data)
    │     │
    │     ├─ 提取数据
    │     │  ├─ 基本信息 (时间段、轮次、职位)
    │     │  ├─ 事件列表 (每个问答)
    │     │  └─ 统计数据 (总数、耗时等)
    │     │
    │     ├─ 计算评分
    │     │  ├─ 遍历所有问答
    │     │  ├─ 根据 LLM 反馈计分
    │     │  └─ 计算平均分
    │     │
    │     ├─ 生成分析
    │     │  ├─ 优点总结 (strengths)
    │     │  ├─ 不足总结 (weaknesses)
    │     │  └─ 改进建议 (suggestions)
    │     │
    │     ├─ 生成 JSON 报告
    │     │  └─ reports/interview_20260322_100000.json
    │     │     {
    │     │       "interview_id": "...",
    │     │       "created_at": "2026-03-22T10:20:00",
    │     │       "round_type": "technical",
    │     │       "position": "java_backend",
    │     │       "overall_score": 7.8,
    │     │       "questions_and_answers": [
    │     │         {
    │     │           "question": "...",
    │     │           "answer": "...",
    │     │           "feedback": "...",
    │     │           "score": 8.0
    │     │         }
    │     │       ],
    │     │       "strengths": ["...", "..."],
    │     │       "weaknesses": ["...", "..."],
    │     │       "suggestions": ["...", "..."]
    │     │     }
    │     │
    │     ├─ 生成 PDF 报告 (可选)
    │     │  └─ 美化格式、添加图表
    │     │
    │     └─ 返回 report_path
    │
    ├─→ DatabaseManager.save_interview()
    │     ├─ INSERT INTO interviews (
    │     │    interview_id, start_time, end_time,
    │     │    duration, report_path
    │     │  )
    │
    ├─→ emit('interview_ended', {success, report_path})
    │
    └─→ 前端
          └─ 跳转到 /report 页面
```

## 8️⃣ 错误处理流程

```
错误发生
    │
    ├─→ 速率限制错误
    │   └─ RateLimiter 返回 False
    │       → emit('error', {'code': 'RATE_LIMIT'})
    │
    ├─→ 输入验证错误
    │   └─ validate_string() 失败
    │       → emit('error', {'code': 'INVALID_DATA'})
    │
    ├─→ LLM 调用错误
    │   ├─ API 超时
    │   ├─ Token 不足
    │   └─ → emit('error', {'code': 'LLM_ERROR'})
    │
    ├─→ 数据库错误
    │   ├─ 连接失败
    │   ├─ 写入失败
    │   └─ → emit('error', {'code': 'DB_ERROR'})
    │
    ├─→ 文件处理错误
    │   ├─ 格式不支持
    │   ├─ 大小超限
    │   └─ → HTTP 400/413
    │
    └─→ 通用服务器错误
        └─ try-catch 捕获
            → logger.error(), emit('error')
```

## 9️⃣ 性能优化关键点

```
【内存使用】
- DataManager 仅保存当前会话数据
- 面试结束后立即导出和清空
- 避免累积问题

【数据库性能】
- 创建索引 (interview_id, user_id, file_hash)
- 定期清理过期数据
- 考虑分表 (按月份)

【LLM API 成本】
- 缓存常见问题的响应
- 复用简历数据 (避免重复加载)
- 控制 token 生成数量

【前端响应时间】
- Socket 事件 < 100ms 返回
- LLM 调用结果 < 2s 返回 (可接受)
- 文件上传可异步处理

【并发处理】
- 数据库支持线程锁
- Socket 可同时处理多个客户端
- 考虑添加连接池
```
